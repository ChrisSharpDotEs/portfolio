Cross-Site Scripting (XSS)

Un atacante inyecta scripts dañinos, generalmente JavaScript, en páginas web vistas por otros usuarios. El script puede robar información de las cookies, secuestrar sesiones de usuario o redirigir a sitios web maliciosos. Existen tres tipos principales de XSS:

  • Reflejado: El script malicioso se inyecta a través de la URL y se "refleja" de vuelta en la página web.
  • Almacenado: El script se almacena en la base de datos del servidor (por ejemplo, en un comentario de un blog) y se ejecuta cada vez que un usuario visita la página.
  • Basado en DOM:Basado en DOM: La inyección ocurre en el navegador del usuario al manipular el Document Object Model (DOM) de la página.

Secuestro de Sesión (Session Hijacking)

El atacante roba la cookie de sesión de un usuario para hacerse pasar por él sin necesidad de conocer su nombre de usuario y contraseña. Una vez que se apodera de la sesión, puede realizar acciones en nombre del usuario legítimo.
